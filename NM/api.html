API Documentation

This system uses a RESTful API to manage books, members, and issue/return records.

  
// Fetch all books
fetch('https://example.com/api/books')  
  .then(response => response.json())  
  .then(data => {  
    console.log(data); // Displays list of all books  
  });  

// Add a new book
fetch('https://example.com/api/books', {  
  method: 'POST',  
  headers: { 'Content-Type': 'application/json' },  
  body: JSON.stringify({  
    title: 'Data Structures',  
    author: 'E. Balagurusamy',  
    isbn: '9781234567890',  
    category: 'Computer Science'  
  })  
})  
  .then(response => response.json())  
  .then(result => console.log('Book added:', result));  

// Issue a book
fetch('https://example.com/api/issue', {  
  method: 'POST',  
  headers: { 'Content-Type': 'application/json' },  
  body: JSON.stringify({  
    bookId: 101,  
    memberId: 2001,  
    issueDate: '2025-10-10',  
    dueDate: '2025-10-20'  
  })  
})  
  .then(response => response.json())  
  .then(result => console.log('Book issued:', result));

The API handles operations like adding, editing, deleting, issuing, and returning books, ensuring real-time synchronization between the database and user interface.
